GFK Dataflow Control Microservice

General Info:

Microservice that connects to one of GFK's servers(or to the mounted drives on those servers) and reads logs of the processes that run on a daily basis for all the countries.

Parameter Info:

  • process = The GFK Application that runs (for example: Porthos, Aramis, Datatrans, RC)
  • isoCode = The country's ISOCode (for example: RO=Romania, AT=Austria, etc.)
  • date = The date from which we want the protocols to be displayed YYYYmmdd format (for example: 20180922)
  • dateFrom = The date from which we want the protocols to be displayed YYYYmmdd format (for example: 20180920) *Used togheter with `dateTo`
  • dateTo = The date until when we want the protocols to be displayed YYYYmmdd format (for example: 20180922) *Used togheter with `dateFrom`
  • filename = The exact name of the log file produced (for example: ScanPlus_dataro_20180920_062118.xml.log)
Route Example Details
/sendLogData/:isoCode/:process /sendLogData/RO/porthos , [data] This is an API that does a POST request to the database with the process name and the JSON data it wants to write to the db.
